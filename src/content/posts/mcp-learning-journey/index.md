The Model Context Protocol is one of those ideas that sounds deceptively simple until you actually try to build with it. This series is a record of that process — working through the O'Reilly material on MCP alongside Claude Code as both a subject of study and an active collaborator.

I'm writing these as I go, which means they're honest about the wrong turns as much as the progress. The goal isn't a polished tutorial. It's a thinking-out-loud record of what it actually takes to understand a technology deeply enough to use it well in a complex, regulated environment like HMCTS.

## What this series covers

Each entry in this journal picks up where the last left off. I'm not jumping ahead — the point is to trace the learning, not to summarise the destination.

The questions I'm carrying into this:

- How does MCP change the way we think about tool integration in AI-assisted workflows?
- What are the practical implications for teams in public sector organisations where governance, auditability and security are non-negotiable?
- Where does the abstraction help — and where does it get in the way?
